## Role

Intelligence OS scores, filters, researches, validates, and prioritizes.

AutoBuilder OS turns approved briefs and architecture documents into build packets for GitHub, Vercel, Supabase, Shopify, content, social, ops, bidding, branding, and automation.

## Cloud-Only Flow

Drive architecture doc -> Intelligence scoring -> AutoBuilder packet -> GitHub issue or file -> Vercel preview or admin task -> review -> deploy or queue.

## Five Minute Build Loop

Every 5 minutes, the system may check for new build packets, new Drive docs, failed automations, pending reviews, and ready tasks.

Allowed autonomous actions:
- inspect Drive files
- draft build packets
- create GitHub issues
- create draft files
- alert Slack
- update Supabase task state
- generate preview plans

Approval required:
- live deployment
- billing action
- public publishing
- customer communication
- product or technical claims
- pricing or warranty changes
- destructive database changes

## Universal System Goal

Use the same operating pattern for website, AI systems, branding, billing, takeoffs, bidding, operations, networking, communications, human and AI resources, social content, engagement, and management.
